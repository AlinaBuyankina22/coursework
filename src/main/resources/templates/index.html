Демо: SQL-инъекции и защита

Демонстрация того, что происходит при конкатенации SQL, и как это устраняется параметризованными запросами и ORM.
Учебный пример

1) Логин (обход аутентификации)

unsafe: SQL собирается строкой → можно “подменить” условие в запросе.
safe: параметры (?) или ORM → ввод не меняет SQL.
Пример для обхода (вставьте в username, пароль — любой):
' OR '1'='1' --

2) Поиск (UNION‑инъекция и утечка данных)

Если поиск сделан небезопасно, можно добавить UNION SELECT ... и вытащить лишние поля (пароль/секретную заметку).
Пример UNION‑payload (вставьте в q):
%' UNION SELECT 1, username, password, secret_note FROM users --
